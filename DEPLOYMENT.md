# SentinelAI Deployment Runbook

This runbook describes the architecture, environment variables, build commands, and deployment steps for launching SentinelAI into production using Cloudflare Pages for the frontend applications and Render for the backend services.

## Architecture & Services

The system comprises four services:

1. **`frontend` (Next.js Dashboard)**
   - **Type:** Static Site
   - **Deployment Target:** Cloudflare Pages
   - **Description:** Admin and dispatch operational dashboard.

2. **`user_frontend` (Next.js Citizen UI)**
   - **Type:** Static Site
   - **Deployment Target:** Cloudflare Pages
   - **Description:** Citizen-facing reporting portal.

3. **`backend` (FastAPI Voice & LLM Service)**
   - **Type:** Web Service
   - **Deployment Target:** Render
   - **Description:** Handles interactive voice and text extraction logic.

4. **`final_endpoints` (Flask ML & Dispatch Service)**
   - **Type:** Web Service
   - **Deployment Target:** Render
   - **Description:** Houses CatBoost models, FAISS historical search, corridor ripple simulator, and SQLite resource tracking.

*Note: The file `backend/app.py` is an unused legacy Flask file and is **not** included in the deployment pipeline.*

---

## Storage & Durability

The `final_endpoints` service relies on a SQLite database (`resources.db`) to track live station resources and dispatch states.
Because Render's default web services use ephemeral file systems that wipe on restart/deploy, a **Render Persistent Disk** has been explicitly configured in `render.yaml` and mounted to `/opt/render/project/src/Resource Intelligence and Historical Operation Module`. This ensures that live resource allocations persist across deployments and restarts.

## Build & Start Commands

**`sentinel-backend-fastapi`**
- **Build:** `pip install -r requirements.txt`
- **Start:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

**`sentinel-final-endpoints`**
- **Build:** `pip install -r requirements.txt`
- **Start:** `gunicorn models:app --bind 0.0.0.0:$PORT`

**`frontend` & `user_frontend`**
- **Framework Preset:** Next.js (Static Export)
- **Build:** `next build`
- **Output Directory:** `out`

## Deployment Order of Operations

### First-Time Deploy

1. **Deploy Backends:**
   Connect your GitHub repository to Render and use the provided `render.yaml` blueprint. This will automatically spin up `sentinel-backend-fastapi` and `sentinel-final-endpoints` (along with the persistent disk).
2. **Note Backend URLs:**
   Once deployed, copy the exact `*.onrender.com` URLs generated by Render.
3. **Update Backend Inter-service Config:**
   In the Render dashboard for `sentinel-final-endpoints`, set `BACKEND_API_URL` to the FastAPI URL.
4. **Deploy Frontends:**
   Connect the repository to Cloudflare Pages (create two projects: one with root directory `frontend/`, one with root directory `user_frontend/`).
   Set the build command to `next build` and output directory to `out`. Add the `*.onrender.com` URLs as environment variables as listed above, and deploy.
5. **Update CORS:**
   Once the Cloudflare Pages generate their `*.pages.dev` URLs, go back to Render and update the `CORS_ALLOWED_ORIGINS` environment variables on both backend services to allow those domains. Restart the Render services.
6. **Redeploy Frontends (if necessary):**
   If any frontend configuration was missed, trigger a redeploy in Cloudflare Pages.

### Subsequent Deploys
Changes pushed to the `main` branch will automatically trigger redeployments on Render and Cloudflare Pages. No manual intervention is needed unless environment variables or external dependencies change.

## Smoke-Test Checklist

After the deployment completes, verify the following:

- [ ] Check `https://<FASTAPI_RENDER_URL>/` loads correctly.
- [ ] Check `https://<FINAL_ENDPOINTS_RENDER_URL>/health` returns `{ "status": "ok" }`.
- [ ] Load the **Dashboard Map** on the admin frontend. Ensure it plots markers correctly.
- [ ] Load the **Analytics** page and verify the heatmap loads data.
- [ ] Load the **Stations & Resources** page and ensure the SQLite persistent disk serves accurate allocations.
- [ ] Open an Incident Detail page and verify the **Recommended Station** and **ML Predictions** panels load.
- [ ] Use the **User Frontend** to submit an incident via the voice interface and confirm the FastAPI backend processes it successfully.
